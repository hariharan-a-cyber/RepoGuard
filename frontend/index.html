Detection pipeline

What runs under the hood

Every scan is deterministic — the same commit yields the same findings. No model in the critical path.

  1. 1

    Rule engine

    Pattern rules for injection, eval/exec, weak crypto, path traversal, NoSQL, and unsafe deserialization across Node.js and Python.

  2. 2

    AST & taint analysis

    Tree-sitter parsing tracks untrusted input from sources to sinks so reported flows are reachable, not guessed.

  3. 3

    Secret detection

    High-signal patterns for API keys, tokens, and credentials, with entropy and placeholder filters to suppress noise.

  4. 4

    Dependency audit

    Manifests checked against the OSV advisory database to surface known-vulnerable packages and their fixed versions.

  5. 5

    Scoring & ranking

    Findings merged, weighted by severity and confidence, and rolled into a single 0–100 risk score.
